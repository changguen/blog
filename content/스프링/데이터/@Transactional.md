- 선언적으로 [[트랜잭션과 영속성 컨텍스트|트랜잭션]]을 거는 방식이다. [[AOP와 프록시의 관계|AOP]]로 동작한다.
- 다음 옵션들을 설정할 수 있다.
## 트랜잭션 전파 속성
**: 트랜잭션이 이미 동작중인 상황에서 현재 트랜잭션이 어떻게 동작할지 설정한다.**
- `REQUIRED` : 있다면 합류, 없다면 생성
- `READUIRES_NEW` : 있다면 일시 정지 후 생성, 없다면 생성
	- 현재 트랜잭션이 롤백되어도 기존 트랜잭션에는 영향이 가지 않는다.
- `SUPPORTS` : 있다면 합류, 없다면 트랜잭션 없이 실행
- `NOT_SUPPORTED` : 있다면 일시 정지, 없다면 트랜잭션 없이 실행
- `MANDATORY` : 있다면 합류, 없다면 예외 (`IllegalTransactionStateException`)
- `NESTED` : 부모가 롤백되면 자식도 롤백, 자식이 롤백되면 부모는 영향 X
## [[트랜잭션 격리 레벨]]
**: 다른 트랜잭션의 변경에 의한 데이터 일관성 레벨을 설정한다.**
## 롤백 규칙
**: 어떤 예외가 발생했을 때 롤백할지 결정한다. (스프링은 기본적으로 런타임 예외에만 롤백한다.)**
- `rollbackFor` : 롤백할 체크예외를 지정할 수 있다.
- `noRollbackFor` : 롤백하지 않을 런타임예외를 지정할 수 있다.
## 읽기 전용
**: 트랜잭션을 읽기 전용으로 설정하여 성능을 최적화한다.**
- JPA 사용시 스냅샷 보관 X / 더티 체킹 X의 최적화가 일어난다.
- 상황에 따라 쿼리를 secondary 서버로 보낼 수도 있다. (`AbstractRoutingDataSource`를 구현해야 한다.)
