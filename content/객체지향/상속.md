: 부모의 특징을 물려받아 재사용하고 확장하는 것
- 코드 구조를 줄이고 계층 구조를 만든다.
- 자바에서는 extends를 통해 구현되었다.
### 장점
- 코드 재사용성
	- 부모의 코드를 재사용 가능하기 때문에, 중복이 줄어든다.
- [[다형성]]의 토대가 된다.
	- 부모의 자리에 자식 클래스가 올 수 있다.
### 단점
- 잘 구현하기 어렵다.
	- IS-A 관계가 아닌 경우에 활용하면 [[SOLID#LSP(리스코프 치환 원칙)|LSP]]를 어길 수 있다.
	- 이 경우 사용하는 측에서 계약이 어긋나므로, 원하는 동작을 제공받을 수 없다.
- 결합도가 높아지며 캡슐화가 깨진다.
	- 자식은 부모의 내부 구현을 알아야 한다. (자식을 구현하기 위해서는 어쩔 수 없다.)
	- 이러면서 자연스럽게 자식은 부모에 강결합되며, 따라서 부모의 수정에 약해진다.
### 내 생각
- '코드 재사용성'을 위해서 상속을 쓰는 것은 위험하다. [[조합]]이라는 뛰어난 도구를 쓰는 것이 낫다.
- 솔직히 말하면 상속의 장점은 '다형성' <- 이거 하나인 것 같은데, 인터페이스와 구현을 활용하는게 상속의 단점을 제거하기 때문에 더 좋은 것 같다.
