: 하나의 객체가 여러 타입을 가질 수 있는 성질이다.
- 구체적으로 말하면, [[추상화]]를 통한 API를 만든 후 해당 API에 수많은 구현체가 들어올 수 있다는 뜻이다.
- 다형성 덕분에 자바에서 오버라이딩을 활용하면, 런타임에 구현체를 동적으로 바꿀 수 있다.
- 다형성이 없다면 분기문이 많아진다.
```java
// AS-IS
class PaymentService {
    public void process(String type) {
        if (type.equals("SAMSUNG")) {
            System.out.println("삼성페이 로직...");
        } else if (type.equals("NAVER")) {
            System.out.println("네이버페이 로직...");
        } else if (type.equals("KAKAO")) { 
            System.out.println("카카오페이 로직...");
        }
    }
}

// TO-BE
class PaymentService {
    public void process(Payment payment) {
        payment.pay(); 
    }
}
```
### 장점
- [[추상화#장점|추상화의 장점]]들과 이어진다.
### 단점
- 직관성이 떨어진다.
	- 실제 구현체가 무엇이 있고, 어느 것이 사용되는지 알기 위해서는 설정 파일이나 구현체를 뒤져야 한다.
