자바에서 '메서드'를 파라미터로 직접 전달할 방법은 없다. 따라서 구현체를 만들고, 그 구현체를 전달하게 된다.
이때, 해당 구현체의 이름은 필요없다. 메서드(동작)이 중요하기 때문이다. 이를 기존의 자바에서는 **익명 클래스**를 통해 수행했다.
```java
Collections.sort(list, new Comparator<String>() {
    @Override
    public int compare(String a, String b) {
        return a.compareTo(b);
    }
});
```
**람다 표현식**은 이 장황한 코드를 짧게 줄여쓰기 위해 등장했다. 동작은 여전히 동일하다. 익명 클래스 전달이다.
```java
Collections.sort(names, (a, b) -> a.compareTo(b));
```
람다 표현식을 통해 우리는 코드를 짭게 줄이고, 함수를 전달하는 듯이 다룰 수 있으며, 가독성을 늘릴 수 있다.
단, 문법에서 보이듯이 메서드를 1개 밖에 전달하지 못한다. 즉, 메서드가 1개뿐인 인터페이스에만 람다 표현식을 넣어줄 수 있다.
