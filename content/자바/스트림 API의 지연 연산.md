스트림 API는 최종 연산(collect, forEach, count, findFirst 등)이 실행되기 전까지 실제 데이터를 처리하지 않는다.
즉, 중간 연산들은 작업을 정의만 해두고, 최종 연산이 이 모든 것을 실행시킨다.
## 핵심 장점
- **루프 퓨전** : 여러 연산들을 하나의 연산처럼 합쳐서 처리하여 효율성을 높인다.
  for 문을 돌 때에는 각 연산에 대해서 원소들을 순회하며 작업한다. (`A원소 연산1`, `B원소 연산1`, `A원소 연산2`, `B원소 연산2` 순서)
  스트림은 반대로 한 원소에 대해서 연산들을 순회한다. (`A원소 연산1,2` ,`B원소 연산1,2`) 따라서 사실상 연산이 합쳐진 것 과 동일하다. (`A원소 연산`, `B원소 연산`)
- **쇼트 서킷** : 더 이상 작업을 이어나가지 않아도 되면 멈춘다.
  예를 들어 최종 연산이 findFirst 인 경우, `A원소 연산` 에서 끝나면 뒷 원소들은 작업하지 않는다.
