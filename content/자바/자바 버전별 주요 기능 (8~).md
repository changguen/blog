- **새로운 Date/Time API (java.time)**
  : 기존에는 `java.util.Date`와 `java.util.Calendar`를 날짜 계산에 사용했다.
  다음 문제가 있었다. : 가변 객체, 0월 부터 시작, 타임존 처리 복잡, 날짜 계산 복잡
  새로운 API에서는 이 문제들이 해결된다. (`LocalDate`, `LocalTime`, `LocalDateTime`, `Instant` 등)
- **디폴트 메서드**
  : 인터페이스를 수정하면 모든 구현체를 수정해야 했다. 이는 엔터프라이즈 코드와 라이브러리 코드의 유지보수를 아주 힘들게 했다.
  따라서 기존 인터페이스 구현체들을 수정하지 않으면서도 새로운 유틸 메서드나 공통적인 기능을 추가하는 방법을 제공하기 위해 **디폴트 메서드**가 탄생했다.

## 17버전 (LTS)
- **switch 에서의 패턴 매칭**
  : switch 문에서 바로 타입 체크하고 해당 타입을 사용할 수 있다.
```java
return switch (obj) {
	case Integer i -> String.format("정수: %d", i);
	case String s -> String.format("문자열: %s", s);
	case Long l -> String.format("Long: %d", l);
	case null -> "null";
	default -> obj.toString();
};

return switch (obj) {
	case String s && s.length() > 5 -> "긴 문자열";
	case String s -> "짧은 문자열";
	case Integer i && i > 0 -> "양수";
	case Integer i && i < 0 -> "음수";
	case Integer i -> "0";
	default -> "기타";
};
```

## 19버전
- **[[가상스레드]]**
- **구조적 동시성**
  : 한 스코프안에서 모든 비동기 작업이 완료되어야 하거나, 하나가 실패하면 모두 멈추는 등, '괄호가 끝나면 모두 끝난다.' 라는 개념을 구조화한다.
  솔직히 동시성 프로그래밍 잘 안해봐서 모르겠다. 흠.
- **레코드 구성 요소 분해**
  : 레코드의 값들을 분해하여 간편하게 접근할 수 있다.
```java
if (obj instanceof Point(int x, int y)) {
	System.out.println("x=" + x + ", y=" + y);
}

switch (response) {
	case ApiResponse(200, String msg, Object data) ->
		System.out.println("성공: " + data);
		
	case ApiResponse(400, String msg, ErrorDetail(String field, String error)) ->
		System.out.println("검증 실패 - " + field + ": " + error);
		
	case ApiResponse(int code, String msg, _) when code >= 500 ->
		System.out.println("서버 오류: " + msg);
		
	default ->
		System.out.println("알 수 없는 응답");
}
```

## 22버전
- **Unnamed Variables & Patterns**
  : 사용하지는 않지만, 문법적으로 필요한 경우 변수명을 `_`로 지어서 명확하게 표현할 수 있다.
```java
catch (NumberFormatException _) { 
    System.out.println("숫자가 아닙니다.");
}

map.forEach((k, _) -> System.out.println("Key: " + k));
```
- **Stream Gatherers**
  : 스트림의 몇개씩 묶어서 처리할 수 있다.
```java
Stream.of(1, 2, 3, 4, 5, 6)
      .gather(Gatherers.windowFixed(3)) // 3개씩 묶는다.
      .toList(); 
// 결과: [[1, 2, 3], [4, 5, 6]]


```
