- 데이터를 디스크에 저장할 때, 임의의 위치에 저장하지 않고, **파일의 끝에 순차적으로 붙여나간다.**
- 덕분에 디스크 기반 저장소임에도, 메모리 기반 저장소 급의 속도를 낸다.
- 이것은 카프카가 기존 데이터를 중간에서 수정하거나 삭제하지 않는 특성에 의거한다.
  카프카가 `Append-only`로 명령하면, OS 수준에서 순차 I/O 처리하게 되는 것이다.
- 참고 : 시간이 지나서 로그가 삭제됨에도 append-only가 보장되는 것은 액티브/클로즈드 세그먼트가 나뉘기 때문이다.
  액티브는 append-only로 순차 I/O 되지만, 클로즈드는 보관 기간이 지나면 통째로 삭제된다.
