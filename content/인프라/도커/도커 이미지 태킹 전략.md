- 여러 성질의 태그들을 달 수 있다.
	- 시멘틱 버저닝 (:1.0.0)
	- 커밋 기반 (:dm32ka)
	- 날짜 기반 (:24.12.19)
	- 환경 기반 (:dev) <- 얘는 덮어쓰기 때문에 버전 추적이 불가능하다.
- 이런 태그들을 혼합하여 사용할 수 있다. ex) 버전+환경+커밋 -> :v1.0.0-prod-qo1d2o
- 아니면 **멀티태깅** 하여 사용할 수 있다.
  이러면 커밋이든 날짜든 버전이든 원하는 것으로 원하는 이미지를 찾을 수 있기 때문에 용이하다.
  그냥 태그를 여러번 달면 된다.
```bash
# 이미지 빌드
docker build -t my-app:build .

# 멀티 태깅
docker tag my-app:build $ECR_REGISTRY/my-app:$VERSION
docker tag my-app:build $ECR_REGISTRY/my-app:${COMMIT_SHA:0:7}
docker tag my-app:build $ECR_REGISTRY/my-app:$ENV
docker tag my-app:build $ECR_REGISTRY/my-app:$ENV-$VERSION

# 모두 푸시
docker push --all-tags $ECR_REGISTRY/my-app
```
