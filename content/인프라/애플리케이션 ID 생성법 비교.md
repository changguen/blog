### SnowFlake
- 64bit (8byte, long)
	- 1 bit : 사용x
	- 41 bit : 밀리초 타임스탬프 (69년치)
	- 10 bit : 머신ID
	- 12 bit : 일련번호 (상위 비트가 같을 경우 순서대로 생성)
- 장점
	- 8bit, long 길이라 성능이 좋다.
- 단점
	- 머신ID 설정이 필요하다.
	  -> 여러 서버 인스턴스가 있을 때, 각각에 대해 ID를 지정해줘야 한다.
### TSID
- 64bit (8byte, long)
	- 42 bit : 밀리초 타임스탬프
	- 10-12 bit : 머신ID (설정하지 않은 경우 서버 실행시 랜덤으로 생성)
	- 10-12 bit : 일련번호 (상위 비트가 같을 경우 순서대로 생성)
- 장점
	- SnowFlake와 거의 비슷하지만, 중간 비트 부분을 편하게 설정할 수 있다.
### 외전: MongoDB ObjectId
- 96bit (12byte)
	- 32 bit : 초 타임스탬프
	- 40 bit : 랜덤 값
	- 24 bit : 일련번호 (상위 비트가 같을 경우 순서대로 생성)
- 장점
	- 서버 설정이 필요없다.
	- 충돌 확률이 0다.
- 단점
	- 데이터 크기가 크다.
	- 초 단위까지만 정렬되며, 그 이후에는 정렬되지 않는다.
