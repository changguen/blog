> **경량형 종합 배치 프레임워크**이다.

- 배경 : 자바 기반 배치 처리 프레임워크가 없어서, 일회성 프로그램이 많았다. 이에 VMWare와 액센츄어가 협력하며 액센츄어가 가지고 있던 고품질의 프레임워크를 다듬은 것이다.
- 스케줄링 프레임워크가 아니다. 스케줄러와 함께 작동하도록 설계되었다.
	- 스프링 배치 : 무엇을 어떻게 실행할 것인가?
	- 스케줄러 : 언제 실행할 것인가?
- 스케쥴(크론식)을 기반으로 실행하거나, 웹으로 트리거하여 실행할 수도 있다.
  딱 배치 관련해서만 프레임워크를 제공하기 때문에 호출부는 유연하게 갈아끼울 수 있다.
- 핵심 패턴
	- Read : 데이터를 조회한다.
	- Process : 데이터를 가공한다.
	- Write : 데이터를 저장한다.
- [[스프링 배치 도메인 언어|핵심 구조]]
	- Job : 배치 처리 과정 전체
	- Step : Job을 구성하는 독립적 단계
	- Chunk : 대량 데이터를 나누어 작업하는 단위
	- 예를 들어,
		- 우아한형제들 정산 작업 : Job
			- 각 사용자의 소비 금액 집계 : Step
				- 1~100번 사용자 : Chunk, 101~200번 사용자 : Chunk ...
			- 각 사용자의 소비 금액에서 우아한형제들 수수료 제외 : Step
			- 각 사용자의 소비 금액에서 라이더 정산 금액 제외 : Step
			- 각 사용자의 나머지 소비 금액을 사장님에게 정산 : Step
		- 우아한형제들 DB 마이그레이션 작업 : Job
- 배치를 사용하는 이유
	- 아주 많은 데이터를 처리하는 도중에 실패하는 상황에 안전하게 처리할 수 있다.
	- 각 배치잡은 딱 1번씩 실행되어 안전하다.
	  -> 이런 작업을 위해 보통 '메타데이터'를 또 따로 저장한다.
	  -> 이것을 [[스프링 배치 메타데이터 테이블|메타데이터 테이블]]에 저장한다.
- 배치 프레임워크를 사용하는 이유
	- 대용량 배치 처리를 쉽게 제공한다. ([[스프링 배치 병렬 처리]])
	- 전체 배치를 나누어 처리하여, 중간 실패에도 유연하게 대응할 수 있도록 한다. ([[스프링 배치 실패 동작]])
	- 결론 : 이런 배치를 쉽게 구현하도록 하여, 도메인 로직 구현에 집중할 수 있다.
