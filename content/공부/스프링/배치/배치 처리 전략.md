- 스프링은 비즈니스 로직을 **표준 빌딩 블록으로 분해하여 설계**할 것을 권장한다.
- **표준 빌딩 블록**
	- 변환 : 외부 시스템 데이터를 표준 형식으로 변환
	- 유효성 검사 : 입출력 레코드의 유효성 검사
	- 추출/업데이트 : 데이터베이스에서 데이터를 읽고 처리한 후 반영
	- 포맷팅 : 처리된 데이터를 전송/인쇄용 표준 형식으로 변환
- 또한 스프링은 배치가 다음과 같은 **처리 옵션**을 가질 수 있다고 한다.
- **배치 처리 옵션**
	- 배치 윈도우 내 일반 처리 : 온라인 사용자가 없는 별도 시간
	- 동시 배치/온라인 처리 : 온라인 사용자와 동시에 데이터를 업데이트할 때 락 사용
	- 병렬 처리 : 여러 배치를 사용하거나, 단일 배치에서 여러 스레드 사용
	- 파티셔닝 : 대규모 작업을 여러 인스턴스로 나누어 동시에 실행
- **파티셔닝 옵션**
	- 고정 분할 : 전체 레코드를 동일한 개수씩 파티셔닝
	- 키 기준 분할 : 특정 ID 범위와 같은 키 값을 기준으로 파티셔닝
	- 동적 구성 : 파일 크기 / 레코드 수 등에 따라 자동으로 파티셔닝
