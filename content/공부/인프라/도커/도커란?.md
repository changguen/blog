> 애플리케이션을 빌드, 배포, 실행하기 위한 오픈소스 플랫폼

- 애플리케이션과 그 실행에 필요한 설정(라이브러리, 종속성)을 [[도커 컨테이너|컨테이너]]라는 표준화된 유닛에 패키징한다.
- **덕분에 어떤 환경에서든 동일하게 실행된다.**
- 장점
	- 일관성 : 개발자 PC의 환경과 동일하게 배포할 수 있어, 쉽고 버그가 덜 발생한다.
	- 빠른 확장성 : 템플릿화 될 수 있기 때문에, 빠르게 수평확장할 수 있다.
- 관련 개념들
	- [[Dockerfile]] : 애플리케이션을 실행시키기 위해 필요한 단계를 기술한 파일
	- [[도커 이미지|이미지]] : Dockerfile을 실행시켜 만든 읽기 전용 템플릿
	- [[도커 컨테이너|컨테이너]] : 이미지를 실행시킨 상태, 실제로 동작하는 프로그램이다.
	- [[docker-compose]] : 여러 이미지를 한번에 띄울 수 있도록 구성하는 파일
- 예시 개발 플로우
	- Dockerfile로 내 애플리케이션을 이미지화 시킨다.
	- 애플리케이션 이미지 + MySQL 이미지 를 한번에 띄울 수 있는 docker-compose 파일을 작성한다.
	- docker-compose 파일을 실행시켜 애플리케이션 컨테이너와 MySQL 컨테이너를 한번에 띄운다.
