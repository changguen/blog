> 컨테이너간 통신 방법

- 기본적으로 `localhost`는 [[도커 컨테이너|컨테이너]] 자기 안을 참조한다. 따라서 다른 컨테이너를 참조하려면 도커 네트워크를 활용해야 한다.
- **같은 네트워크 안의 컨테이너들은 컨테이너 이름으로 서로를 찾을 수 있다. (DNS 지원)**
  ex) `http://myapp:8080/user/123`
- [[도커 컴포즈]]를 사용하면 기본적으로 선언된 컨테이너들끼리 통신할 수 있는 네트워크를 만들어준다.
- 기본 run 명령어를 사용하면 기본적으로 제공되는 `bridge` 라는 이름의 네트워크를 사용하게 된다. (드라이브 종류는 bridge이다.)
  주의! 기본 bridge 네트워크는 컨테이너 이름으로 서로를 찾을 수 없다.(DNS 미지원) 그냥 명목상으로 연결만 된다고 생각하는게 올바르다.
- **네트워크 종류**
	- bridge : 같은 네트워크의 컨테이너간 연결
	- host : 호스트 네트워크 직접 사용
	- none : 네트워크 미사용
