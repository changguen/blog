- [[B+Tree 인덱스]] 같은 B-Tree 계열은 데이터를 정렬된 상태로 유지하며, **읽기 성능에 최적화**된 인덱스이다.
- LSM-Tree 인덱스는 디스크의 순차 쓰기 속도가 빠른 것을 이용하여 **쓰기 성능을 최적화**한다.
- 작동 원리
	- MemTable : 모든 쓰기 요청이 메모리 상의 정렬된 구조에 저장된다.
	- SSTable : MemTable이 가득 차면 디스크에 SSTable 형태로 한번에 쓴다. (불변)
	- Compaction : 여러 개의 SSTable이 존재할 때, 중복을 제거하고 정렬하여 새로운 큰 SSTable을 만든다.
- 하지만 읽기 시에 여러 SSTable을 확인해야 할 수 있기 때문에 **읽기가 느리다.**
