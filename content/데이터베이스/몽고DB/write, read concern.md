## Write Concern
**: 데이터를 얼마나 안전하게 저장할지에 대한 옵션이다.**
- `0` : 저장한지와 상관없이 응답한다.
- `1` (default) : primary 에 저장되면 응답한다.
- `2` : primary + seconday 1개에 저장되면 응답한다.
- `majority` : 과반수 노드에 저장되면 응답한다.
- `3` : 모든 노드에 다 쓰면 응답한다.
주의! 여기서 `저장`은 메모리에 저장된다는 뜻이다. 메모리가 아니라 디스크에 저장되었을 때를 기준으로 하려면 `journal`값을 만져야 한다.
### Journal
**: 데이터를 어디에 저장한 것을 기준으로 할지에 대한 옵션이다.**
- `true` : 디스크에 저장해야 한다.
- `false` : 메모리에만 저장해도 된다.
즉, `w:3 j:false` 조합은 '모든 노드의 메모리에 저장되면 OK'라는 뜻이다.
## Read Concern
**: 얼마나확정된 데이터를 읽을지에 대한 옵션이다.**
primary에서 읽으면 상관없지만, secondary에 있는 데이터는 롤백될 수도 있는 미확정 데이터이기 때문이다.
- `local` (default) : 쿼리한 노드에 있는 데이터를 응답한다. (롤백될 수 있는 데이터일 수도 있다.)
- `available` : local과 비슷하나, 샤딩중 남은 찌꺼기인 `Orphaned documents`도 읽을 수 있다.
- `majority` : 과반수 노드가 인정한 데이터만 응답한다. (절대 롤백 안된다.)
- `linearizable` : primary에만 읽기가 가능하다.
